This file records external projects, sources, and methodology references that influence LaunchLens. It should be updated whenever a new outside project is added to the product, documentation, demo, or developer workflow.

## Attribution Rule For Future Updates

When adding a third-party project:

1. Add the project name, repository URL, license, and license URL here.
2. State whether LaunchLens copied code, adapted code, embedded assets, or only referenced methodology.
3. Add visible product or documentation links where users can inspect the source.
4. Update `CHANGELOG.md` in the same commit.
5. Re-run tests and submission validation before pushing.
